# v1.0.16 - 2026-06-17

## Fixed

- use map-based insert in `AppInfoCache.GetOrCreateByAppID` auto-create path so an empty RTP is persisted as `''` instead of falling back to the database column default (`95`) when the column still has `DEFAULT 95`

## Note

`Select().Create()` does not reliably persist empty string RTP values when `app_info.rtp` still has `DEFAULT 95` in MySQL. Prefer map inserts or run the `app_info.rtp` migration from v1.0.15.
