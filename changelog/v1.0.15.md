# v1.0.15 - 2026-06-17

## Changed

- remove the default RTP value (`95`) from `models.AppInfo`; the `rtp` column now uses `default:NULL`, so merchants can be created without an implicit RTP
- when `AppInfoCache` auto-creates `app_info` via `GetOrCreateByAppID`, explicitly persist the `rtp` column with `Select` to avoid database default values overriding an empty RTP

## Migration

If `app_info.rtp` still has `DEFAULT 95` in MySQL, run:

```sql
ALTER TABLE `app_info`
  MODIFY COLUMN `rtp` varchar(191) DEFAULT NULL COMMENT '默认rtp';
```
